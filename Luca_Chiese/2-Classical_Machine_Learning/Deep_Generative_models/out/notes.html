MLE and MAP

Maximum Likelihood Estimation (MLE) and Maximum a Posteriori Estimation (MAP).

In general our objective is to estimate the parameters. There are two methods to do so: MLE and MAP.
The PDF is our model, how do we estimate its parameters? If the pdf is gaussian we would use the mean and standard deviation.

MLE and MAP allow to estimate the parameters θ\thetaθ of a stastistical model based on the observed data XXX. But the difference is that the MLE tries to maximize the conditional probability of XXX given θ\thetaθ, p(X∣θ)p(X|\theta)p(X∣θ).

p(X∣θ)p(X|\theta)p(X∣θ) = how likely it is to observe this data if the parameters are θ\thetaθ

MLE

MLE tries to find θ\thetaθ that maximize the likelihood p(X∣θ)p(X|\theta)p(X∣θ), i.e., θ^MLE=argmax⁡θp(X∣θ)\hat{\theta}_{MLE} = \text{arg} \max_\theta p(X|\theta)θ^MLE​=argmaxθ​p(X∣θ).

Are we making any hypothesis on the prior distribution on θ\thetaθ? NO, the probability p(θ)p(\theta)p(θ) is not present in the formula.

How do we find θ\thetaθ that maximize the likelihood p(X∣θ)p(X|\theta)p(X∣θ)?

MAP

MAP tries to find the parameter θ\thetaθ that maximizes the posterior probability. From a bayesian perspective, it includes a priori probability P(θ)P(\theta)P(θ) that reflects our belief about the parameter values. This improves the optimization process.

In altre parole, il prior è una distribuzione di probabilità che esprime ciò che crediamo sul parametro θ\thetaθ prima di osservare i dati. Può rappresentare conoscenze precedenti, intuizioni, o semplicemente preferenze su certi valori dei parametri.

P(θ∣X)=P(X∣θ)P(θ)P(X)∝P(X∣θ)P(θ) P(\theta | X) = \frac{P(X | \theta) P(\theta)}{P(X)} \qquad \propto P(X | \theta) P(\theta) P(θ∣X)=P(X)P(X∣θ)P(θ)​∝P(X∣θ)P(θ)

In this way, the MLE optimization gets modified in the MAP optimization as below

What happens if log⁡P(θ)\log P(\theta)logP(θ) is constant?
This means that the prior P(θ)P(\theta)P(θ) does not favor any particular value of θ\thetaθ: it is a flat uniform distribution.

log⁡P(θ)\log P(\theta)logP(θ) plays the role of a regularization term and can help in reducing overfitting. However, if it is constant we are not regularizing anything and, since the parameters start from a random initialization, the average behavior of many parameters might seem to “cancel out” or stabilize, due to the law of large numbers.

If log⁡P(θ)\log P(\theta)logP(θ) is constant, then MAP reduces to MLE because the constant doesn't affect the location of the maximum.

Pros and Cons

  • MLE:
    • simpler to implement
    • more prone to overfitting if there is limited data and no regularization/prior is used
  • MAP:
    • can help reduce overfitting by imposing a prior -> often acts like a regularization term
    • useful when we have domain knowledge or want to encode certain parameter restrictions

Discriminative vs Generative

Difference: when we want to calculate the conditional probability of the output given the input, in supervised (discriminative) models we only focus on the prior probability of the output, in generative models we focus on the prior probability on the input and the conditional probabiliy on the data.

To calculate the conditional probability P(Y∣X)P(Y|X)P(Y∣X), first estimate the prior probability P(Y)P(Y)P(Y) and the likelihood probability P(X∣Y)P(X|Y)P(X∣Y) from the data provided. Our aim in generative model is to obtain the posterior

posterior=prior×likelihoodevidence⇒P(Y∣X)=P(Y)P(X∣Y)P(X) . posterior = \frac{prior \times likelihood}{evidence} \Rightarrow P(Y|X) = \frac{P(Y) P(X|Y)}{P(X)} \ . posterior=evidenceprior×likelihood​⇒P(Y∣X)=P(X)P(Y)P(X∣Y)​ .

To make things clearer, let's define the variables:

  • (X)( X)(X): the input data, e.g., an image, an email, or a feature vector describing a sample.
  • (Y)( Y)(Y): the output label, e.g., "cat", "not spam", "class A".

Discriminative models directly model P(Y∣X)P(Y \mid X)P(Y∣X), i.e., the probability of a label given the input data.
Generative models, instead, model the joint probability P(X,Y)=P(Y)⋅P(X∣Y)P(X, Y) = P(Y) \cdot P(X \mid Y)P(X,Y)=P(Y)⋅P(X∣Y), and then derive P(Y∣X)P(Y \mid X)P(Y∣X) via Bayes' theorem:

P(Y∣X)=P(X∣Y)⋅P(Y)P(X)P(Y \mid X) = \frac{P(X \mid Y) \cdot P(Y)}{P(X)} P(Y∣X)=P(X)P(X∣Y)⋅P(Y)​

This means that generative models also capture how the data ( X ) is distributed within each class ( Y ), which allows them to generate new samples or understand the data distribution better.

Variational AutoEncoders (VAEs)

AutoEncoder

An autoencoder is a feed-forward neural net whose job is to take an input xxx and predict xxx. To make this task non-trivial, we need to add a bottleneck layer whose dimension is much smaller than the input.

The encoder compresses the input data into a lower-dimensional representation, known as the latent space. This compressed representation (or code vector) is then passed to the decoder, which reconstructs the original input as closely as possible.

But why do this? The idea is to capture the most important features of the input in a simplified form. If the neural network can reconstruct the input accurately from this lower-dimensional code, it means that the code captures the essence of the data, what’s truly important, while discarding less relevant details. This process is similar in spirit to PCA (Principal Component Analysis), but unlike PCA, neural networks can capture complex, nonlinear patterns. By analyzing the latent space, we can gain insight into the key factors that define the data, as learned by the network.

Simplest:

  • 1 hidden layer
  • linear activations
  • simple squared error loss L(x,x~)=∣∣x−x~∣∣2\mathcal{L}(x, \tilde{x}) = || x - \tilde{x} ||^2L(x,x~)=∣∣x−x~∣∣2

This network computes x~=UVx\tilde{x} = UVxx~=UVx, which is a linear function. If K≥DK \geq DK≥D, we can choose UUU and VVV such that UV=IUV = \mathbb{I}UV=I. This isn't very interesting.

A linear autoencoder with squared error loss is equal to PCA!!!!

Then, why to autoencode?

  • dimensionality reduction
  • denoising
  • representation learning

Pros:

  • Simple and fast to implement.
  • Effective at compressing high-dimensional data into lower-dimensional representations.
  • Can learn useful features for downstream tasks like classification or clustering.
  • Useful for denoising and dimensionality reduction, even with limited data.

Limitations:

  • The latent space is not regularized, which means there is no guarantee that similar inputs will have similar encodings.
  • There is no probabilistic interpretation, so we cannot sample meaningfully from the latent space.
  • The latent space (K units) is often discontinuous and unstructured: it’s just a collection of vectors without smooth transitions or organization.
  • Cannot generate new data in a principled way (unlike VAEs or GANs).

Variational AutoEncoder

Main limitation: autoencoders and deep autoencoders (more hidden layers) do not define a distribution, they do not generate anything, they reproduce!
If the latent space is not continuous, the reconstruction loss is high, and we can sample correctly only in correspondence of training data.

Can we make the latent space continuous? From which we can sample? This is the motivation behind Variational AutoEncoders.

For Variational AutoEncoders, as in AutoEncoders:

  • input xxx, hidden layer output zzz, reconstruction x~\tilde{x}x~
  • encoder is a neural network with parameters (weights and biases) θ\thetaθ
  • input: 28×2828 \times 2828×28 pixel photo →784\to 784→784-dimensional representation
  • hidden layer len(z)len(z)len(z) is much smaller than 784784784
  • decoder is a neural network with parameters (weights and biases) ϕ\phiϕ

But in addition:

  • the encoder is characterised by a probability density function qθ(z∣x)q_\theta(z|x)qθ​(z∣x) generally a Gaussian
  • efficient compression with this distribution
  • from this distribution, we can sample noisy values of zzz
  • the decoder is characterised by a pdf pϕ(x∣z)p_\phi(x|z)pϕ​(x∣z) from which we try to reconstruct x~\tilde{x}x~

Information from the original 784784784-dimensional vector cannot be perfectly transmitted, because the decoder only has access to less information in the form of a less-than-784784784-dimensional vector zzz.
How much information is lost?

KL Divergence

The Kullback-Leibler (KL) divergence quantifies how much information is lost.

Let log⁡pϕ(x∣z)\log p_\phi(x|z)logpϕ​(x∣z) be the reconstruction log-likelihood, that tells me how effectively we have learned to reconstruct an input image xxx, given its latent representation zzz.
The loss function is a negative log-likelihood with a regularization term (for each iii-th sample)

li(θ,ϕ)=−Ez∼qθ(z∣xi)[log⁡pϕ(xi∣z)]+KL(qθ(z∣xi)∣∣p(z)) .l_i (\theta, \phi) = - \mathbb{E}_{z \sim q_\theta(z|x_i)}[\log p_\phi(x_i|z)] + \mathbb{KL}(q_\theta(z|x_i) || p(z)) \ . li​(θ,ϕ)=−Ez∼qθ​(z∣xi​)​[logpϕ​(xi​∣z)]+KL(qθ​(z∣xi​)∣∣p(z)) .

The total loss is the sum of losses from iii to NNN.

  • The first term is the reconstruction loss :Ez∼qθ(z∣xi)[log⁡pϕ(xi∣z)]: \mathbb{E}_{z \sim q_\theta(z|x_i)}[\log p_\phi(x_i|z)]:Ez∼qθ​(z∣xi​)​[logpϕ​(xi​∣z)] is the expectation taken w.r.t. the encoder's distribution, qθ(z∣xi)q_\theta(z|x_i)qθ​(z∣xi​), over the representations, pϕ(xi∣z)p_\phi(x_i|z)pϕ​(xi​∣z). It tells how much the decoder is capable to decode the encoder but from the encoder's point of view. We use the log⁡\loglog because it stabilizes numerical values, simplifies optimization, and has strong informational meaning.
  • The second term is the Kullback-Leibner Divergence: summarize the difference (divergence) between the probability function of the encoder qθ(z∣xi)q_\theta(z|x_i)qθ​(z∣xi​) and that of the decoder p(z)p(z)p(z). So, it quantifies how much information is lost when using qqq to represent ppp.

In the Variational Autoencoder, the latent variables zzz are supposed to come from a prior distribution, which is specified to be a normal distribution with zero mean and unit variance, i.e. p(z)=N(0,1)p(z) = \mathcal{N}(0,1)p(z)=N(0,1). The encoder learns a distribution q(z∣xi)q(z∣x_i)q(z∣xi​) for each input, and we want this distribution to be close to the prior. This is enforced by a KL divergence term in the loss. If the encoder tries to produce latent codes that deviate too much from this standard normal shape, the model is penalized (it will receive a penalty in the loss). This regularization helps keep the latent space well-behaved and consistent, making it easier to interpolate between data points or sample new, realistic data.

Probabilistic interpretation of VAE

A Variational Autoencoder (VAE) defines a probabilistic model over observed data x\mathbf{x}x and latent variables z\mathbf{z}z. The joint distribution can be factorized as:

p(x,z)=p(x∣z) p(z)p(\mathbf{x}, \mathbf{z}) = p(\mathbf{x} \mid \mathbf{z}) \, p(\mathbf{z}) p(x,z)=p(x∣z)p(z)

where:

  • x\mathbf{x}x: observed variable (e.g., input data)
  • z\mathbf{z}z: latent variable (hidden representation)

  • Generative process: for each datopoint iii:

    • draw zi∼p(z)→z_i \sim p(z) \tozi​∼p(z)→ drawn from a prior
    • draw xi∼p(x∣z)→x_i \sim p(x|z) \toxi​∼p(x∣z)→ drawn from a probability (Bernoulli distributed for black/white images)

  • Inference (testing after training) -> Goal: infer good values of zzz given xxx, or to calculate p(z∣x)p(z|x)p(z∣x).
    How can we find good values of zzz? Use the Bayes' Theorem

    p(z∣x)=p(x∣z)p(z)p(x) p(z|x) = \frac{p(x|z) p(z)}{p(x)} p(z∣x)=p(x)p(x∣z)p(z)​

We already know p(x∣z)p(x|z)p(x∣z) and p(z)p(z)p(z) (we sample from the network), but what about p(x)p(x)p(x)? p(x)p(x)p(x) is the prior probability of the data. How to compute it?

The value of p(x)p(x)p(x) indicates how well the generative model can explain or generate the observed data xxx. However, computing p(x)p(x)p(x) directly is often intractable due to the complexity of the integral. To address this difficulty, an approximate distribution q(z∣x)q(z \mid x)q(z∣x) is introduced, and a quantity called the Evidence Lower Bound (ELBO) is maximized, which provides a lower bound to log⁡p(x)\log p(x)logp(x).

We could marginalize the latent variables: p(x)=∫p(x∣z)p(z)dzp(x) = \int p(x|z)p(z) dzp(x)=∫p(x∣z)p(z)dz. However, this integral takes time to be evaluated since one needs to evaulate it over all configurations of zzz (i.e., evaluate this integral for all zzz that have been generated in the encoder).

We know that qλ(z∣x)q_\lambda(z|x)qλ​(z∣x) (from the encoder) has a parameter λ\lambdaλ which indexes the family distribution. If qqq were gaussian it would be the mean and variance of each data point, λi=(μxi,σxi)\lambda_i = (\mu_{x_i}, \sigma_{x_i})λi​=(μxi​​,σxi​​). Then the vector λ=(λ1,λ2,…,λN)\lambda = (\lambda_1, \lambda_2, \ldots, \lambda_N)λ=(λ1​,λ2​,…,λN​) replaces the parameters vector θ=(θ1,θ2,…,θN)\theta = (\theta_1, \theta_2, \ldots, \theta_N)θ=(θ1​,θ2​,…,θN​) and each parameter θi\theta_iθi​ gets substituted by a mean and standard deviation. So, each parameter is a gaussian distribution that allows us to cover better the latent space.

From now on we call qqq Variational Posterior.

How can we know how well our variational posterior qλ(z∣x)q_\lambda(z|x)qλ​(z∣x) approximates the true p(z∣x)p(z|x)p(z∣x)? Again, use the KL Divergence

KL(qλ(z∣x)∣∣p(z∣x))=Eq[log⁡qλ(z∣x)]−Eq[log⁡p(x,z)]+log⁡p(x) , \mathbb{KL}(q_\lambda(z|x) || p(z|x)) = \mathbb{E}_q[\log q_\lambda(z|x)] - \mathbb{E}_q[\log p(x,z)] + \log p(x) \ , KL(qλ​(z∣x)∣∣p(z∣x))=Eq​[logqλ​(z∣x)]−Eq​[logp(x,z)]+logp(x) ,

This formula measures how different the approximate distribution qλ(z∣x)q_\lambda(z \mid x)qλ​(z∣x) is from the true posterior p(z∣x)p(z \mid x)p(z∣x). In other words, it quantifies how much information is lost when using a simpler distribution to approximate the true one, which quantifies the amount of information that is lost in the data compression. But how do we again compute p(x)p(x)p(x)?
Define the ELBO (Evidence Lower Bound)

ELBO(λ)=Eq[log⁡p(x,z)]−Eq[log⁡qλ(z∣x)] , ELBO(\lambda) = \mathbb{E}_q[\log p(x,z)] - \mathbb{E}_q[\log q_\lambda(z|x)] \ , ELBO(λ)=Eq​[logp(x,z)]−Eq​[logqλ​(z∣x)] ,

and isolating

log⁡p(x)=ELBO(λ)+KL(qλ(z∣x)∣∣p(z∣x)) , \log p(x) = ELBO(\lambda) + \mathbb{KL}(q_\lambda(z|x) || p(z|x)) \ , logp(x)=ELBO(λ)+KL(qλ​(z∣x)∣∣p(z∣x)) ,

from the Jensen's inequality one can state that KL≥0\mathbb{KL} \geq 0KL≥0 (because I cannot gain information compressing data). So, given that KL≥0\mathbb{KL} \geq 0KL≥0 and log⁡p(x)≥0\log p(x) \geq 0logp(x)≥0, instead of minimizing KL\mathbb{KL}KL we can maximize ELBOELBOELBO. In this way we avoid the computation of the prior probability of the data, log⁡p(x)\log p(x)logp(x).

ELBO

ELBOi(λ)=Eqλ(z∣xi)[log⁡p(xi,z)]−KL(qλ(z∣xi)∣∣p(z)] , ELBO_i(\lambda) = \mathbb{E}_{q_\lambda(z|x_i)}[\log p(x_i,z)] - \mathbb{KL}(q_\lambda(z|x_i) || p(z)] \ , ELBOi​(λ)=Eqλ​(z∣xi​)​[logp(xi​,z)]−KL(qλ​(z∣xi​)∣∣p(z)] ,

the first term is the reconstruction loss and the second term is the compression loss.

Let's remember now NN. The final step is:

  1. Parametrize the approximate posterior qθ(z∣x,λ)q_\theta(z|x, \lambda)qθ​(z∣x,λ) with an inference network (encoder)
  2. Parametrize the likelihood p(x∣z)p(x|z)p(x∣z) with a generative network (decoder) that takes latent variables zzz and outputs parameters to the data distribution pϕ(x∣z)p_\phi(x|z)pϕ​(x∣z)
  3. ELBOi(θ,ϕ)=−li(θ,ϕ)ELBO_i(\theta, \phi)= - l_i(\theta, \phi)ELBOi​(θ,ϕ)=−li​(θ,ϕ)

Summary

  1. Model parameters (often denoted θ,ϕ\theta, \phiθ,ϕ):
    • θ\thetaθ typically refers to the generative model parameters, e.g. the decoder network pθ(x∣z)p_\theta(x|z)pθ​(x∣z).
    • ϕ\phiϕ typically refers to the inference (recognition) network parameters, e.g. the encoder network qϕ(z∣x)q_\phi(z|x)qϕ​(z∣x).
    • These parameters are global to the entire dataset. You train one set of θ,ϕ\theta, \phiθ,ϕ that applies to every datapoint.
  2. Variational parameters (often denoted λ\lambdaλ):
    • In classical variational inference, for each datapoint x(i)x^{(i)}x(i), you might introduce a local variational distribution qλ(i)(z)q_{\lambda^{(i)}}(z)qλ(i)​(z) with its own parameters λ(i)\lambda^{(i)}λ(i).
    • These local parameters λ(i)\lambda^{(i)}λ(i) describe the approximate posterior distribution for a specific datapoint x(i)x^{(i)}x(i).
    • In a generic variational inference setting (e.g., for Bayesian model), you'd explicitly optimize these local λ(i)\lambda^{(i)}λ(i) alongside any global parameters.
    • In practice, VAEs don't store a separate set of parameters λ(i)\lambda^{(i)}λ(i) for each datapoint. Instead, they use a neural network (the encoder) parametrized by ϕ\phiϕ which outputs the mean μϕ(x)\mu_\phi(x)μϕ​(x) and variance σϕ(x)\sigma_\phi(x)σϕ​(x) (or whichever parameter one needs) for the approximate posterior q(z∣x)q(z|x)q(z∣x).
    • Formally, each datapoint x(i)x^{(i)}x(i) has its own latent distribution parameters (i.e. local λ(i)\lambda^{(i)}λ(i)) computed by the encoder, but we do not treat them as free variables we store and optimize individually. We instead learn the ϕ\phiϕ that maps x(i){x^{(i)}}x(i) to λ(i){\lambda^{(i)}}λ(i).

In other words:

  1. θ,ϕ→\theta, \phi \toθ,ϕ→ global neural network weights for the decoder/encoder. These are explicitly optimized during training and shared across all data.
  2. λ→\lambda \toλ→ local variational parameters per datapoint. In a standard VAE, they are produced on the fly by plugging xxx into the encoder network qϕ(z∣x)q_\phi(z|x)qϕ​(z∣x). We do not keep a separate λ\lambdaλ variable to optimize for each datapoint; the encoder network ϕ\phiϕ handles that functionally.

Hence, the pratical difference is:

  • Model parameters θ,ϕ\theta, \phiθ,ϕ are a fixed set of weights for the entire model (shared across the entire dataset).
  • Variational parameters λ\lambdaλ (in a strict variational-inference sense) are the parameters of the approximate posterior distribution for each individual datapoint. In a VAE, these local parameters are never individually stored or optimized; rather, they are generated by the encoder (whose weights ϕ\phiϕ are optimized globally).

That is why we often talk about global and local parameters in a variational framework. In typical VAE implementations, we only see the global θ,ϕ\theta, \phiθ,ϕ in code, but conceptually each datapoint has its own local latent distribution characterized by the mean μϕ(x)\mu_\phi(x)μϕ​(x) and variance σϕ(x)\sigma_\phi(x)σϕ​(x), which can be seen as its local λ\lambdaλ.

In the formula x~=UVx\tilde{x} = UVxx~=UVx, UUU and VVV do not contain parameters in the sense of a neural network which has an activation function, but they are neurons that are characterized by a probability density function, each of them having its own mean and standard deviation. Then each global θ\thetaθ parameter has its own local λ\lambdaλ which, in a gaussian sense, is represented by the mean and standard deviation.

Generative Adversarial Networks (GANs)

Two NN: generator + discriminator

  1. Generator takes random noise as input and tries to produce real data. We optimize a model which takes in input some random noise, into the generator, and tries to generate fake images.
  2. Discriminator tries to distinguish fake from real data. It is trained out of real and fake images and tries to discriminate which are real and which are fake.

Generator and discriminator are, in some sense, "enemies", unlike the encoder and decoder in a VAE where if I improve the encoder, that is, I encode better the data with less information loss, then the decoder will work better. Here, if the generator is amazing at producing images is a problem for the discriminator because it will not be able to distinguish real from fake images.

This is called Adversarial Game: one tries to fool the other!!

Two steps are involved:

  1. generate a batch of samples (Generator)
  2. update the weights based on correct/wrong (Discriminator)

Some definitions:

We are trying to do a simultaneous min max optimization, that is minimize w.r.t. θg\theta_gθg​ and maximize w.r.t. θd\theta_dθd​. The first term, the likelihood of true data, is the probability the discriminator actually disciminates. The second term is the likelihood of generated data; when this term is zero there is no loss meaning that the generated data are exactly what the discriminator is expecting.

This optimization is a problem: it is a saddle-point optimization.

This is the reason why the training of GANs is very complex, much more complex than the VAEs.

Training

Example of a trainig loop

  • αd→\alpha_d \toαd​→ learning rate of the discriminator
  • αg→\alpha_g \toαg​→ learning rate (not exactly) of the generator

In the training loop the generator is updated after the discriminator. If the generator improves before the discriminator and starts associating specific noise patterns with specific fake images, the discriminator will only learn to recognize those specific patterns and not generalize well to new ones.

Historical issues

  • Vanishing gradient: the discriminator becomes "too good" and the generator gradient vanishes.
  • Non-convergence: the generator and discriminator oscillate without reaching an equilibrium.
  • Mode collapse: the generator distribution collapses to a small set of examples.
  • Mode dropping: the generator distribution doesn't fully cover the data distribution.

Vanishing gradient

It happens when the discriminator is amazing. The generator updates its parameters using gradients from the discriminator, but:

  • If the discriminator becomes too good too quickly, it can easily distinguish fake data from real data.
  • As a result, the discriminator’s output for fake data becomes close to 000, and the gradient it provides to the generator becomes very small (i.e., vanishes).
  • This means GGG can't learn, because it’s not receiving meaningful updates.
  • When D(G(z))≈0D(G(z)) \approx 0D(G(z))≈0, the derivative of log⁡(1−D(G(z)))\log(1 - D(G(z)))log(1−D(G(z))) becomes near to zero --> vanishing gradient.

The fact that the discriminator is amazing is not useful because our goal is not to discriminate something but to have a generative model. In this case the parameters θd\theta_dθd​ and θg\theta_gθg​ get not updated because the gradient is vanishing.

How to fix this?

  • possibilities for non-saturating objectives: if the discriminator is not perfect, we can define some objective function that converges to some non-constant number that depends only on the generator. We add something like E[log⁡G]\mathbb{E}[\log G]E[logG] to the min⁡max⁡\min \maxminmax formula, but this would imply that to reduce this term to zero the generator will generates only one image and quickly this possibility becomes useless. This could lead to mode-collapse.
  • use non-saturating loss for GGG: minimize log⁡D(G(z))\log D(G(z))logD(G(z)) instead of log⁡(1−D(G(z)))\log(1 - D(G(z)))log(1−D(G(z))).
  • restrict capabilities of D\mathcal{D}D (structurally!!) (limit the number of neurons, use not complex cnn, ...)
  • balance learning DθdD_{\theta_d}Dθd​​ and GθgG_{\theta_g}Gθg​​

The last two solutions are more advisable than the first one.

Non-convergence

DθdD_{\theta_d}Dθd​​ and GθgG_{\theta_g}Gθg​​ are often non-convex, so the saddle point is complex to find. Training updates undo each other.

The left picture shows convergence under Adam optimization. We see that in order to converge to the minimum we have to move in only one direction. The right picture shows convergence under saddle-point optimization, in order to converge to the saddle point we have to move in two directions.

Mode-collapse

It happens when the generator focuses on producing a limited set of data patterns that deceive the discriminator. This could happen as a consequence of using non-saturating objective functions to solve the vanishing gradient problem.

The model becomes fixated on a few dominant modes in the training data and fails to capture the full diversity of the data distribution.

We see that GAN gets fixated on generating only 111 and 999.

The solution is provided by WGAN (Wasserstein GANs) in which the Wasserstein distance is used instead of the cross-entropy. The Wasserstein-1 distance (or Earth-Mover's distance) is continuous and differentiable almost everywhere with respect to the generator's parameters. This means that when I take the derivative of the loss function during the training, the fact that it is continuous and differentiable implies that there are no discontinuities. This imply that even when generated data has little overlap with the real distribution, the W-distance provides non-zero gradient. This prevents the gradient from vanishing and, as a consequence, prevents the mode-collapse.

Diffusion Models (DFs)

Based on the diffusion process -> two steps: forward + backward.

All the generative models we have seen (VAE, GAN, DF) have the concept of mutual challenge between generation and discrimination.

While in GAN we were providing random noise to a generator to generate an image and train the parameters to associate noise to image, and the discriminator had the job of taking an image and say whether it is fake or not. Here, in DF, we gradually add small amounts of (gaussian) noise to the initial image, x0x_0x0​, and the backward process consists in just removing noise from the noisy image.

While in VAE and GAN we can define layers, the initial image is processed by layer1, layer1-output is processed by layer2, and so on. The final output depends on the matrix multiplications taking place at each later. The number of matrix multiplications is equal to the number of layers. This could be pretty challenging. In the case of DF we proceed as in a Markov Chain (t+1t+1t+1 depends only on ttt).

Same formula as VAE, we just add a TTT because this process has to be done pairing each time-step between process ppp and process qqq and iterating for all time-steps. In practice there are hundreds or thousands of time-steps, depending on the task. So, this optimization will take a lot of time.

The difference between VAEs and DFs is that in DFs the variational parameter is prompted, unlike VAEs where the variational parameter is λ\lambdaλ.

The variational parameter for DFs is the prompt we write to describe the image we want to be generated by the model. The text is embededd in numerical form in order for the model to undestand it, via some kind of text embedding techinique. The model learns to associate the description we have provided (the prompt desctibing, for example, the background we wish) to some skeleton model (for example the pose of a man) and tries to generate the image. Then, it tries to combine two inputs, the prompt and the skeleton model to generate the final image (man in a specific pose with the background we have described).

Applications

VAEs are capable to both training and predicting very fast, things that stable diffusion models do not do. Secondly VAEs, thank to the variational parameter λ\lambdaλ, have the capability of covering all the latent space. However, the diversity of the generated images is much less than the one obtained by Diffusion Models.

VAEs do not have high quality samples even after a lot of epochs. DFs reach high quality samples but they need a large amount of time.
